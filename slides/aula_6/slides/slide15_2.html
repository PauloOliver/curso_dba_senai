Breve História dos SGBDs Pt.2

  • Arquivos (1960): Antes dos SGBDs, os dados eram guardados em arquivos simples (ex: .txt, .csv). Pouca organização, muita duplicação e difícil manutenção.
  • Modelo Hierárquico (1968): Estrutura em árvore (pai → filhos).
    Bom para dados bem estruturados, mas pouco flexível. Exemplo: IMS da IBM.
  • Modelo Relacional (1970): Criado por Edgar F. Codd. Dados organizados em tabelas (linhas e colunas).
    Muito mais flexível e com SQL para consultas. Até hoje é o modelo mais usado (MySQL, PostgreSQL, Oracle, SQL Server).
  • NoSQL (2009): Surgiu com a necessidade de lidar com grandes volumes de dados e alta escalabilidade.
    Estruturas mais flexíveis: chave-valor, documentos, colunas, grafos. Usado em sistemas distribuídos e aplicações modernas (MongoDB, Cassandra, Redis).
