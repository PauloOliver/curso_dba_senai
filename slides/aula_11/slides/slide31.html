CHECK: Validação de Dados

O CHECK é uma restrição de integridade que você coloca em uma coluna (ou na tabela) para validar os dados no momento da inserção ou atualização.

Ele define uma condição lógica que deve ser verdadeira. Se alguém tentar inserir um valor que não satisfaça a condição, o SQLite recusa a operação e retorna erro.

Sintaxe básica:

CHECK em coluna:


CREATE TABLE Funcionarios (
    FuncionarioID INTEGER PRIMARY KEY,
    Nome TEXT NOT NULL,
    Salario DECIMAL(10,2) CHECK (Salario > 0)
);

Aqui, o SQLite só permite que o Salario seja maior que 0.

Exemplo de erro:


INSERT INTO Funcionarios (Nome, Salario) VALUES ('Ana', -1000);
-- Resultado: erro, CHECK constraint failed

CHECK em tabela inteira:


CREATE TABLE Pedidos (
    PedidoID INTEGER PRIMARY KEY,
    Quantidade INTEGER,
    Preco DECIMAL,
    CHECK (Quantidade * Preco <= 10000)
);

Nesse caso, a condição verifica uma expressão envolvendo múltiplas colunas.

Exemplo: você não pode inserir Quantidade = 50 e Preco = 500 porque 50*500 = 25000 > 10000.

Exemplos comuns de uso:

Caso Exemplo de CHECK
Salário positivo Salario DECIMAL CHECK(Salario > 0)
Estoque não negativo Estoque INTEGER CHECK(Estoque >= 0)
Sexo válido Sexo TEXT CHECK(Sexo IN ('M','F'))
Nota entre 0 e 10 Nota REAL CHECK(Nota >=0 AND Nota <=10)
