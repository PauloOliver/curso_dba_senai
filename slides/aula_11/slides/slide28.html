Índices: Com e Sem Índice

Situação sem índice

Imagine uma tabela Clientes com 1 milhão de registros:


CREATE TABLE Clientes (
  ClienteID INTEGER PRIMARY KEY,
  Nome TEXT,
  Email TEXT
);

E você roda a consulta:


SELECT * FROM Clientes WHERE Nome = 'Maria';

O que o banco faz sem índice?

  • Ele precisa ler linha por linha da tabela inteira (processo chamado full table scan).
  • Ou seja, olha o registro 1, depois o 2, depois o 3… até o 1.000.000 para ver onde Nome = 'Maria'.
  • Isso pode ser muito lento em tabelas grandes.

Situação com índice

Agora criamos um índice na coluna Nome:


CREATE INDEX idx_clientes_nome ON Clientes (Nome);

➡ O que acontece agora?

  • O banco cria uma estrutura de dados interna (geralmente uma árvore B-Tree ou similar) com os nomes já organizados.
  • Isso funciona como um índice de livro: quer achar “Maria”? Vai direto na letra M e encontra rapidinho.
  • O banco não precisa mais ler todas as linhas — ele usa o índice para ir direto nas posições corretas.
