Monitorando o Servidor com SHOW STATUS

O MySQL permite acompanhar o desempenho do servidor em tempo real usando variáveis de status:

1️⃣ Listando todos os status disponíveis

SHOW GLOBAL STATUS;

Exemplo de saída:

Variable_name Value
Uptime86400
Threads_connected12
Threads_running3
Questions12500
Slow_queries8
Innodb_buffer_pool_reads1050
Innodb_buffer_pool_read_requests50000
Com_commit2000
Com_rollback5

Interpretação:

  • Uptime → servidor rodando há 1 dia (86400s).
  • Threads_connected e Threads_running → quantas conexões e queries ativas.
  • Slow_queries → possíveis problemas de performance.

2️⃣ Filtrando apenas variáveis do InnoDB

SHOW GLOBAL STATUS LIKE 'Innodb%';

Exemplo de saída filtrada:

Variable_name Value
Innodb_buffer_pool_reads1050
Innodb_buffer_pool_read_requests50000
Innodb_row_lock_waits12

Interpretação prática:

  • Innodb_buffer_pool_reads → leituras que precisaram ir ao disco. Número alto → buffer pool pequeno.
  • Innodb_buffer_pool_read_requests → total de leituras solicitadas ao cache.
  • Innodb_row_lock_waits → quantas vezes transações aguardaram lock de linha.

3️⃣ Exemplo de análise real

Para monitorar performance de um e-commerce, você poderia rodar:

SHOW GLOBAL STATUS LIKE 'Slow_queries';
SHOW GLOBAL STATUS LIKE 'Threads_running';
SHOW GLOBAL STATUS LIKE 'Innodb_buffer_pool_reads';

Exemplo de saída:

Variable_name Value
Slow_queries45
Threads_running10
Innodb_buffer_pool_reads8000

Análise:

  • 45 queries lentas → otimizar índices ou queries.
  • 10 queries rodando simultaneamente → aceitável, mas monitorar se crescer.
  • 8000 leituras do disco → considerar aumentar o InnoDB Buffer Pool para reduzir I/O.
