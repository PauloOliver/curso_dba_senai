Ferramentas de Monitoramento Internas ao MySQL

O MySQL oferece comandos e esquemas internos valiosos para monitoramento:

Comandos SHOW STATUS:

  • SHOW GLOBAL STATUS;: Contadores globais do servidor (Uptime, Queries, Slow_queries, Threads_connected, etc.).
  • SHOW SESSION STATUS;: Contadores para a sessão atual.
  • Exemplo: SHOW GLOBAL STATUS LIKE 'Innodb_buffer_pool_read%';
  • No MySQL Workbench: Muitos desses status são exibidos na aba "Server Status".

INFORMATION_SCHEMA:

  • Um banco de dados que contém metadados sobre o SGBD.
  • Tabelas úteis:
    • PROCESSLIST: Sessões ativas e suas queries.
    • TABLES: Informações sobre tabelas (tamanho, engine).
    • STATISTICS: Informações sobre índices.
    • INNODB_TRX, INNODB_LOCKS, INNODB_LOCK_WAITS: Detalhes sobre transações e locks do InnoDB.
  • No MySQL Workbench: Acessível via editor SQL.

1️⃣ Listando todos os status disponíveis

Você pode ver todas as variáveis de status assim:

SHOW GLOBAL STATUS;

Isso retorna uma lista enorme com duas colunas:

  • Variable_name → Nome da variável, como 'Uptime', 'Threads_connected', 'Slow_queries', etc.
  • Value → Valor atual da variável.

Filtrando apenas variáveis do InnoDB:

SHOW GLOBAL STATUS LIKE 'Innodb%';

Algumas variáveis comuns retornadas:

  • Innodb_buffer_pool_read_requests
  • Innodb_buffer_pool_reads
  • Innodb_row_lock_waits

2️⃣ Exemplos de variáveis comuns

Variável O que mede
Uptime Tempo que o servidor está rodando em segundos
Threads_connected Quantas conexões estão ativas
Threads_running Quantas queries estão sendo processadas ao mesmo tempo
Questions Total de queries enviadas ao servidor
Slow_queries Total de queries que ultrapassaram long_query_time
Innodb_buffer_pool_reads Leituras de dados que precisaram ir ao disco
Innodb_buffer_pool_read_requests Total de leituras solicitadas ao buffer pool
Com_commit Total de commits de transações
Com_rollback Total de rollbacks
